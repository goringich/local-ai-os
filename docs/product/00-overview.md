# Product

LOCAL AI OS makes one private Linux/Codex workflow observable and bounded: relevant context, approved write scope, verification, and a report for the owner.

## Public repository and delivery

The public repository contains this product site, the proof contract, reviewed technical documentation, release checks, and a GitHub Pages build. It is not a downloadable copy of a customer's runtime or credentials.

- Public: product documentation, schemas, safe proof data, and site build logic.
- Delivered during implementation: reviewed workstation configuration, scoped policies, integration choices, and acceptance artifacts.
- Local-only: customer source, secrets, prompts, raw responses, runtime state, and private reports.

## Capability status

Stable means a documented, bounded practice exists. Experimental means it needs owner review per installation. Planned means it is not delivered by cloning this repository. Unavailable means no claim is made.

- No universal autonomy promise.
- No unrestricted shell bridge.
- No claim that a public-site clone installs LOCAL AI OS.
