# Acceptance

Acceptance is the owner-facing proof that the agreed workflow and boundaries were actually checked.

## Handoff evidence

An acceptance report lists declared artifacts, scope, completed checks, known limitations, recovery route, and next action. A failed or unavailable measurement stays visible.

- Verification evidence is mandatory for a successful run.
- Acceptance compares delivery with the agreed scope, not with an autonomy claim.
- The owner retains the report and operating decisions.

## Status

The report shape and verification requirement are stable. Customer-specific acceptance criteria are agreed before an implementation starts.
