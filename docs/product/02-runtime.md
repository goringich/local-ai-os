# Runtime

Runtime is a role map, not a single product dependency: local models, a human UI, guarded assistant paths, and deterministic checks can be assembled around the existing workstation.

## Roles and health

Ollama provides local model execution where selected. Open WebUI is a human-facing local UI and knowledge surface. OpenClaw and OpenHarness are guarded task/check surfaces where adopted. Their health is assessed separately from the static product-site build.

- Model routing is configuration and installation-specific.
- Fallback is an explicit operator decision, not a silent cloud escalation.
- A missing service is reported as unavailable, not simulated in the portal.

## Status

The runtime integration is experimental because exact components, ports, and resource limits are approved during implementation. The safety and reporting boundaries are stable.
