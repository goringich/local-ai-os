# Integrations

Integrations are selected to extend the existing local-first contour, not to introduce a parallel always-on agent stack.

## Installation choices

An implementation can connect approved local model services, repositories, notes, and operator surfaces. Each connector receives a named owner, data classification, least-privilege boundary, and rollback route.

- No integration receives unrestricted access by default.
- Customer-specific connectors are not published in this repository.
- Unapproved cloud transfer is unavailable by design.

## Status

Integration catalogues are planned. The review process and public/private boundary are stable.
